Guide

Requêtes

URL

Toutes les requêtes se font sous la forme, avec la méthode GET :

Racine + path + « / »

Avec :

Racine = https://mobilite.elycee.rhonealpes.fr/mobilite/

Les headers

Obligatoires à chaque demande

X-Kdecole-Auth = clé (« » en l’absence de clé) (Voir Activation)

X-Kdecole-Vers = version de l’app (« 3.1.15 » lors de la rédaction de cette documentation)

Réponses

Headers

Ces headers sont présents pour chacune des réponses

Spécifiques à l’API

Date : Date d’envoi de la réponse (string en lisible, formaté en US)

X-Kdecole-Expire : expiration des information (?)

X-Kdecole-Date : (apparemment) Date en timestamp en ms

Classiques

Accept-Ranges, Content-Length, Content-Type, Set-Cookie

Body

L’api utilise des objets Json

Ses réponses contiennent toujours (sauf précision, les variables sont des string) :

errmsg : objet d’erreur (null en cas de réussite)

            tempsvalide : utilité indéfinie

            message : message à destination de l’utilisateur, formaté en HTML

            code : code (http ?) correspondant (corrélation non établie)

            titre : titre de l’erreur, à destination de l’utilisateur

            type : défini l’erreur (peut prendre comme valeur DROP (le compte doit etre supprimé), INFO (erreur d’importance mineure), BLOCK (erreur d’importance majeur, l’application ne peut continuer))

Les autres possibilités sont précisées dans les Endpoint qu’elles concernent.

Note

Les timestamp sont sur l’UTC, en ms

Références

Activation

Sert à récupérer la clé d’api, lors de la première connexion

Requête

Path

activation/id/code

Variables

id est l’identifiant mobile

code est le « code d’activation »

Note

Ces strings doivent impérativement être en minuscule (=vérification de l’entrée utilisateur nécessaire)

Réponse

Contenu de l’objet

errmsg

authtoken est un string qui contient la clé d’authentification à l’API (clé)

success (bool) prends la valeur true quand l’opération a réussi

En cas de réussite

Headers

Habituels sauf X-Kdecole-Expire et X-Kdecole-Date 

Body

{

"errmsg" : null,

"success" : true,

"authtoken" : <Clé>

}

En cas d’échec

Headers

Habituels sauf X-Kdecole-Expire et X-Kdecole-Date 

Body

{

"errmsg" : null,

"success" : false,

"authtoken" : null

}

Info utilisateur

Sert à récupérer des informations sur l’utilisateur nécessaires pour pointer des informations sur d’autres Endpoint

Requête

Path

infoutilisateur

Requêtes secondaires

Path

Servent à changer l’établissement ou l’élève sélectionné

infoutilisateur/ideleve/ideleve

 

infoutilisateur/idetablissement/idetablissement

Variables

ideleve est l’identifiant élève

idetablissement est l’identifiant établissement

Note

Ces deux commandes retournent les mêmes informations que l’autre

Réponse

Contenu de l’objet

Errmsg

etabs (array d’objets) listes des établissements

uid identifiant de l’établissement (possible de le transformer en int)

            active (bool) définit si l’établissement est actif (?)

            permissions liste des permissions (séparés par un espace)

            nom nom de l’établissement

idEtablissementSelectionne identifiant de l’établissement sélectionné (quoi ? Une api non restfull ? (C’est pourtant comme restfull qu’elle est vendue…))

eleves (la rétro-ingénierie, avec les moyens actuels, ne permet pas de résoudre le contenu de cet array)

xiti (objet) système d’analytique ?

            codeProfil ?

            idEtab ?

            idXiti ?

            idCollectivite ?

            idProjet ?

            idPlateforme ?

protection ?

nom nom de l’élève

type type d’utilisateur ?

idEleveSelectionne (la rétro-ingénierie, avec les moyens actuels, ne permet pas déduire avec certitude l’utilité de cette variable)

Exemple

Headers

Habituels

Body

{

    "errmsg" : null,

    "etabs" : [

        {

            "uid" : "<id de l’établissement>",

            "active" : true,

            "permissions" : "lecture-actu info-actus messagerie lecture-cdt lecture-taf cdt-calendrier cdt-travaux",

            "nom" : "<nom de l’établissement>"

        }

    ],

    "idEtablissementSelectionne" : ""<id de l’établissement> ",

    "eleves" : null,

    "xiti" : {

        "codeProfil" : "<?>",

        "idEtab" : "<?>",

        "idXiti" : "<?>",

        "idCollectivite" : "<?>",

        "idProjet" : "<?>",

        "idPlateforme" : "<?>"

    },

    "protection" : "",

    "nom" : <nom de l’élève>,

    "type" : 2,

    "idEleveSelectionne" : null

}

Contenu de l’objet

Errmsg

listeJourCdt (array d’objets jour)

            listeSeances (array d’objets séance)

    aFaire (array d’objets info-séance)

            type type de l’info-séance (prends comme valeur connues « Cours », « Devoir à la maison », « Exercices », « TP » et « Autre »)

            date (int) timestamp en ms de l’info-séance (heure du cours pour un cours, jour du devoir pour un devoir (afaire), jour où le devoir a été donné pour un devoir (arendre), etc…)

            titre titre de l’info-séance

    matiere nom de la matière

    flagModif ?

    salle nom de la salle (et aussi de la classe)

    aRendre (array d’objet info-séance)

    titre ?

    enSeance (array d’objet info-séance)

    hfin (int) timestamp de fin de séance en ms

    idSeance (int) id de séance

    hdeb (int) timestamp de debut de séance en ms

    flagActif (bool) ?

    motifModif ?

            date (int) timestamp en ms du jour concerné

                    "flagModif" : false,

                    "salle" : "<nom de la salle>",

                    "aRendre" : null,        //il n’y a rien a rendre pour la sénce suivante

                    "titre" : "",

                    "enSeance" : [

                        {

                            "type" : "Cours",              //ceci est bien un objet dit info-séance

                            "date" : <timestamp du debut du cours>,

                            "titre" :"<titre du cours>",

                            "uid" : <id du cours>

                        }

                    ],

                    "hfin" : <timestamp de la fin du cours>,

                    "idSeance" : <identifiant de la séance>,

                    "hdeb" : <timestamp du debut du cours>,

                    "flagActif" : true,

                    "motifModif" : ""

                },

                          …

                         "date" : <timestamp du matin de la journée concerné>

          ],

                …

}

Variables

Réponse

Contenu de l’objet

errmsg

titre titre de l’info-séance

url ? (Semble rediriger vers le cahier de texte)

type type d’info-séance

flagRealise (bool) marqué comme réalisé

isFaitModifiable (bool) possible de changer l’état de réalisation

matiere matière concerné par cette info-séance

codeHTML code HTML du contenu (il est conseillé de l’afficher)

flagTravailAFaire (bool) travail à faire

{

    "type": "Exercices",

    "pjs": [],

    "isFaitModifiable": true,

    "matiere": "ANGLAIS LV1",

    "codeHTML": "<beaucoup d’HTML qui rends la doc trop longue >",

    "flagTravailAFaire": true

}

Contenu de l’objet

modeReponse mode de réponse (prends les valeurs « false », « TOUS »)

expediteurActuel (objet) définit l’expéditeur actuel (?)

            libelle nom (lisible) de l’expéditeur

            id identifiant de l’expéditeur

participations (array) objets de participations/messages

            dateEnvoi (int) timestamp en ms du message

            corpsMessage message formaté en HTML

            typeMessage type du message (valeur connue « TEXT_RICH »)

            premieresLignes extrait du message

            pjs (array) pièces jointes ?

            libelleObjet ?

            redacteur (objet) auteur du message

    libelle nom lisible de l’auteur

    id identifiant de l’auteur

nbParticipations (int) nombre de message ?

type type de conversation (peut prendre comme valeur : « COMMUNICATION_MASSE », « DISCUSSION_MODEREE »)

participants (array) objets de participants

            libelle nom (lisible) du participant

            id identifiant du participant

etatLecture (bool) true si le dernier message est lu

signalable (bool) possible de le signaler

objet objet du dernier message

etat ? (Seule valeur connue « ACTIVE »)

id identifiant de la conversation

pieceJointe (bool) false en cas d’absence de pièce jointe

isExpediteurInitial (bool) ?

premieresLignes premiers caractères du dernier message = aperçu

expediteurInitial (objet) définit l’expéditeur initial (?)

            libelle nom (lisible) de l’expéditeur

            id identifiant de l’expéditeur

Exemple

{

            "pjs": [],

            "libelleObjet": null,

            "redacteur": {

                "libelle": "<nom de l’auteur>",

                "id": "<id de l’auteur>"

            },

            "id": <id du message>

        }

    ],

    "nbParticipations": 1,

    "type": "DISCUSSION_MODEREE",

    "participants": [

        {

            "libelle": "<nom de l’auteur>",

            "id": "<id de l’auteur>"

        }
